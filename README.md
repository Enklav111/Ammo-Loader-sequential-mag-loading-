# Ammo Loader — поочерёдная зарядка магазинов

Мод для S.T.A.L.K.E.R. Anomaly (GAMMA), расширяющий **Anomaly Magazines Redux**. Позволяет заряжать магазин патронами разных типов в заданном циклическом порядке: тип A (N патронов), тип B (M патронов), тип C (K патронов), … и так до заполнения магазина или исчерпания патронов в инвентаре.

## Возможности

- Контекстное меню «зарядить поочерёдно» на любом магазине в инвентаре игрока (не в контейнере и не у NPC).
- Горячая клавиша `U` (по умолчанию) — **умное поведение**:
  - Если нет открытых интерфейсов → открывает real-time оверлей настройки пресетов. Игра не ставится на паузу.
  - Если инвентарь открыт и курсор наведён на магазин → запускает поочерёдную зарядку этого магазина.
- **Отладка (MCM, секция внизу, выкл по умолчанию):** клавиша дампа (по умолчанию `MMB`) — вывод в сообщения порядка патронов. Работает только если включена **«Отладка»**:
  - В инвентаре: магазин под курсором.
  - Вне инвентаря: магазин в оружии в руках; модификатор в MCM — `Shift` / `Ctrl` / `Alt` / **без модификатора** (одна клавиша дампа). Оружие — только с поддержкой Magazines Redux.
  - Сначала порядок **выстрела** (не более 10 групп, затем `...`), отступ, затем заголовок **«Остаток:»** и с новой строки по одному типу патрона с суммарным количеством в магазине (блок с янтарно-оранжевой подсветкой).
- Для каждого калибра — свой пресет: до 6 типов патронов, каждому задаётся количество подряд (1–99) перед переключением на следующий тип.
- В пресете поддерживаются: добавление типа, удаление, изменение порядка (↑/↓), изменение количества (+/−).
- Если пресет для калибра не настроен — зарядка идёт в алфавитном порядке по совместимым с магазином патронам (по 1 патрону каждого типа).
- Пресеты сохраняются **в каждом отдельном сейве** (через `save_state`/`load_state`).
- Иконки патронов в UI рендерятся из игровых текстур (как в инвентаре).
- Совместимость с системой «испорченных» патронов GAMMA (`_bad` / `_verybad`).

## Два режима зарядки (MCM-опция)

| Режим | Поведение |
|---|---|
| **По патрону** (по умолчанию) | Как в оригинальном Magazines Redux: патроны грузятся по одному с задержкой и звуком из ltx (`load_delay`). Счётчик магазина `X/Y` обновляется после каждого патрона. |
| **Мгновенный** | Магазин заполняется целиком в один кадр (удобно для тестов / быстрой игры). |

### Прерывания постепенной зарядки

Зарядка автоматически прерывается в трёх случаях:

1. **Закрытие инвентаря** — ловится через `GUI_on_hide("UIInventory")`.
2. **Перетаскивание патрона на магазин** — Magazines Redux видит занятый action-slot, вызывает свой `do_interrupt()`, мы это ловим через `magazines.action_in_progress()` и прерываемся. Перетаскиваемый патрон при этом НЕ загружается (таково поведение Mags Redux).
3. **Магазин покинул инвентарь игрока** — проверка `mag:parent():id() == AC_ID` на каждом тике.

При прерывании уже загруженные патроны сохраняются в магазине, игроку показывается сообщение о числе заряженных патронов (и при необходимости под ним строки с разбивкой по типам патронов, цветом как в отладочном «Остаток») либо «Зарядка прервана».

## MCM-опции

Мод появляется в списке MCM как **Ammo Loader**.

| Опция | По умолчанию | Назначение |
|---|---|---|
| Клавиша пресетов / зарядки под курсором | `U` | Smart-хоткей: пустой HUD → оверлей пресетов; инвентарь + курсор на магазине → поочерёдная зарядка. |
| Мгновенная зарядка | Выкл | ВКЛ = залить магазин целиком. ВЫКЛ = зарядка по патрону с задержкой. |
| Игнорировать испорченные патроны | Вкл | ВКЛ = `_bad`/`_verybad` не видны в UI и не заряжаются. |
| **Отладка** | Выкл | ВКЛ = активны клавиша дампа и модификатор для дампа; ВЫКЛ = дамп не обрабатывается (клавиша `U` / зарядка — как обычно). |
| Клавиша дампа (отладка) | `MMB` (339) | Только при ВКЛ «Отладка». Инвентарь: магазин под курсором. В руках: с модификатором (см. ниже) или **без модификатора** в MCM. |
| Оружие в руках: модификатор (отладка) | `Shift` | `Без` / `Shift` / `Ctrl` / `Alt`. Для дампа в руках вне инвентаря. `Без` — только клавиша дампа. |

Секция **Отладка** в MCM расположена **внизу** (после основных опций), отделена линией.

### Ограничение перебинда клавиши дампа

MCM блокирует бинд на `mouse1/2/3` (левая/правая/средняя кнопки мыши) через UI-виджет выбора клавиши. При этом в поле **default** эти значения принимаются нормально — поэтому средняя кнопка мыши (DIK 339) работает «из коробки», но при перебинде через MCM доступны только клавиатурные клавиши и `mouse4+`.

### Горячие клавиши в открытом инвентаре

Пока открыт `UIInventory`, движок не шлёт нажатия в глобальный `on_key_press` — события обрабатывает `ui_inventory.UIInventory:OnKeyboard`, как в моде `Mags_Unload_and_Eject`. Ammo Loader подключается к той же цепочке: обёртка вызывается при `WINDOW_KEY_PRESSED`, читает предмет под курсором через `hover.bag` / `hover.idx` и `CC[bag]:GetObj(idx)`.

## Алиасы калибров

Некоторые игровые секции патронов технически относятся к «разным» калибрам, хотя на практике это один и тот же боеприпас. Мод объединяет их жёстко прошитой таблицей:

| Источник | Назначение | Комментарий |
|---|---|---|
| `12x76` | `12x70` | 12-калибр: обычные и магнум-гильзы в одной группе. Все 12-калиберные (buck, zhekan, bull, dart) смешиваются. |
| `pkm` | `7.62x54` | Ленты ПКМ/ПКП (`ammo_pkm_100`) показываются вместе с патронами СВД. |

Для `pkm` стоит флаг «не добавлять в дефолтный пресет»: лента не попадёт в автоматический пресет магазина даже если магазин её принимает — её нужно добавить вручную через UI.

Добавить новый алиас — см. `ИНСТРУКЦИЯ_ПО_РАСШИРЕНИЮ.md`.

## Зависимости

- **Обязательно:** Anomaly Magazines Redux (иначе мод отключается при старте с сообщением в лог).
- **Опционально:** MCM (для настройки клавиши и режимов). Без MCM используются значения по умолчанию.

## Точки интеграции с Anomaly Magazines Redux

Мод использует **только публичные API** Magazines Redux, без monkey-patching:

| API | Для чего |
|---|---|
| `magazine_binder.is_magazine(obj)` | Определение магазина |
| `magazine_binder.get_magazine_caliber(mag)` | Список совместимых патронов |
| `magazine_binder.get_data(id)` / `set_data(id, data)` | Чтение/запись содержимого магазина |
| `magazine_binder.get_basetypes_by_ammo_type(sec)` | Проверка что патрон имеет хоть один совместимый магазин |
| `magazines.get_mag_time(section, false)` | Задержка между патронами (из MCM Mags Redux) |
| `magazines.action_start_magload()` | Взять action-slot Mags Redux |
| `magazines.action_in_progress()` | Детект прерывания (кто-то сбросил action-state) |
| `magazines.do_interrupt_magload()` | Освободить action-slot по окончании |
| `magazines.inventory_refresh(2)` | Обновить счётчик `X/Y` в инвентаре (прямой вызов, не `timed_refresh` — тот дебаунсит на 0.5s и пропускает обновления при частых тиках) |
| `stack.push(mag_data.loaded, ammo)` | Добавить патрон в стек магазина |

## Технические нюансы

- **Русская локализация в CP1251.** Файл `rus/st_sequential_load.xml` обязан быть закодирован в Windows-1251. Если править через современный редактор, он почти наверняка сохранит в UTF-8 → в игре будут кракозябры или знаки вопроса. После любой правки русского XML перекодировать:
  ```powershell
  $p='gamedata\configs\text\rus\st_sequential_load.xml'
  $t=[System.IO.File]::ReadAllText($p,[System.Text.Encoding]::UTF8)
  [System.IO.File]::WriteAllText($p,$t,[System.Text.Encoding]::GetEncoding(1251))
  ```
  Символы, которых нет в CP1251 (например em-dash `—`, типографские кавычки `«»` иногда), превратятся в `?`. Использовать обычный дефис `-` и ASCII-кавычки.

- **MCM auto-hint.** В `sequential_load_magazine_mcm.script` **нельзя** указывать `hint = "..."` для опций, потому что MCM сам подставит префикс `ui_mcm_` к значению → будет двойной префикс. Оставлять поле hint пустым, тогда MCM автосгенерирует `<group_id>_<option_id>` и найдёт наши XML-строки.

- **Название в списке MCM.** Строка `ui_mcm_menu_seqload` — это именно то, что видит игрок в левом списке модов MCM. Строка `ui_mcm_seqload` — заголовок правой панели. Эти два разных ключа.

- **Пресеты привязаны к сейву, не глобальные.** Логика в `save_state`/`load_state` кладёт таблицу `presets` в `mdata`. Разные сейвы — разные пресеты.

- **Обновление счётчика магазина.** Используется `magazines.inventory_refresh(2)`, НЕ `timed_refresh()`. Последний дебаунсит обновление на 0.5s и при тиках каждые 0.4s никогда не успевает сработать до конца зарядки — счётчик застывал на 0/20 до полного заполнения.
