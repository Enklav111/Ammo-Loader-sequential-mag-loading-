# Инструкция по расширению Ammo Loader

Документ описывает, как **добавлять новые калибры**, **группировать патроны в алиасах** и **вносить локальные правки** без поломки мода. Все правки локальные и не требуют перекомпиляции — игра подхватит их при следующем запуске (или при `/devrestart`, если установлен соответствующий мод-дебагер).

---

## 1. Как работает определение калибра

В файле `gamedata/scripts/sequential_load_magazine.script` есть две функции:

```lua
local function extract_caliber_raw(ammo_section)
    -- ammo_5.45x39_fmj  -> "5.45x39"
    -- ammo_12x70_buck   -> "12x70"
    -- ammo_pkm_100      -> "pkm"
    -- ammo_gauss        -> "gauss"
    ...
end

function extract_caliber(ammo_section)
    -- то же самое, но потом применяет CALIBER_ALIASES
end
```

Правило простое: **калибр = всё, что между `ammo_` и следующим `_`**. Если подчёркиваний после `ammo_` нет (как у `ammo_gauss`) — берётся всё до конца. Затем применяется таблица алиасов.

### Никакого ручного регистра калибров не нужно

Если вы добавили в игру новый патрон вида `ammo_9x33_hp`, он будет автоматически:
- отнесён к калибру `9x33`;
- появится в выпадающем списке калибров UI, если хотя бы один магазин в игре его принимает;
- попадёт в дефолтный пресет для магазинов, которые его принимают.

Ничего править в моде НЕ надо — достаточно что патрон существует в `weapon_ammo.ltx` и как минимум один магазин указал его в `mag_ammo_list` / прочих списках совместимости Anomaly Magazines Redux.

## 2. Как добавить новый алиас (объединение двух калибров)

Алиасы нужны когда технически разные калибры по факту — один и тот же боеприпас, и игрок хочет видеть их в одной группе в UI.

**Примеры что уже есть:**
- `12x76 → 12x70` (магнум-гильзы 12 калибра)
- `pkm → 7.62x54` (ленты ПКМ к патронам СВД)

### Пошагово

Откройте `gamedata/scripts/sequential_load_magazine.script`, найдите блок:

```lua
local CALIBER_ALIASES = {
    ["12x76"] = "12x70",
    ["pkm"]   = "7.62x54",
}
```

Добавьте новую строку:

```lua
local CALIBER_ALIASES = {
    ["12x76"]   = "12x70",
    ["pkm"]     = "7.62x54",
    ["9x33"]    = "9x32",     -- ← новый алиас
}
```

**Формат:**
- **Ключ** (слева) = сырой токен калибра (то, что даёт `extract_caliber_raw` для ammo секции, см. пункт 1).
- **Значение** (справа) = в какой «канонический» калибр перенаправить.

После этого:
- Патроны `ammo_9x33_*` будут отображаться в UI в группе калибра `9x32`.
- Пресет сохраняется под ключом `9x32` (а не `9x33`), так что один пресет работает для обоих.
- Алгоритм зарядки продолжает использовать реальные section names — ничего не ломается.

### Как узнать сырой токен калибра для нового патрона

1. Найдите секцию патрона в `weapon_ammo.ltx` (обычно в `configs/items/weapons/`), например:
   ```ini
   [ammo_9x33_hp]:ammo_base
   ```
2. Отрежьте префикс `ammo_`, возьмите всё до следующего `_`:
   ```
   ammo_9x33_hp  →  9x33
   ```
3. Именно `9x33` и есть сырой токен, который надо использовать как ключ в `CALIBER_ALIASES`.

## 3. Как пометить алиас как «не добавлять в дефолтный пресет»

Иногда нужно: патрон должен быть ДОСТУПЕН для добавления в пресет (показываться в выпадашке «Добавить»), но НЕ попадать в автоматический алфавитный пресет. Пример — ленты ПКМ в пресете для СВД.

В том же файле `sequential_load_magazine.script` есть блок:

```lua
local HIDDEN_FROM_DEFAULT = {
    ["pkm"] = true,
}
```

Добавьте сырой токен калибра (тот же, что в `CALIBER_ALIASES`) со значением `true`:

```lua
local HIDDEN_FROM_DEFAULT = {
    ["pkm"]  = true,
    ["9x33"] = true,    -- ← новый скрытый источник
}
```

Что это меняет:
- Патроны `ammo_9x33_*` **по-прежнему** отображаются под канонич. калибром `9x32` в UI.
- При вызове «зарядить поочерёдно» на новом магазине, дефолтный пресет НЕ будет содержать 9x33-патронов.
- Игрок сможет вручную добавить их через кнопку «Добавить тип патрона», и тогда они будут заряжаться.

---

## 4. Как удалить алиас

Просто удалите строку из `CALIBER_ALIASES` (и при наличии — из `HIDDEN_FROM_DEFAULT`).

Побочный эффект: **существующие сохранённые пресеты могут «сломаться»**. Если у игрока был пресет под ключом `7.62x54`, содержащий патроны и СВД, и ПКМ (через алиас), после удаления алиаса `pkm → 7.62x54`:
- Патроны СВД останутся в пресете `7.62x54` — ок.
- Патроны ПКМ (`ammo_pkm_100`) окажутся «сиротами» в пресете, но `filter_preset_for_mag` их молча отфильтрует при зарядке.
- Игрок сможет настроить отдельный пресет для калибра `pkm` заново.

Ничего не ломается в рантайме, но игрок потеряет свою настройку ПКМ.

## 5. Как добавить новую MCM-опцию

Скажем, нужно добавить чекбокс «Автоматически закрывать меню после сохранения пресета».

### Шаг 1. Зарегистрировать опцию в MCM-скрипте

В `gamedata/scripts/sequential_load_magazine_mcm.script` найдите блок `on_mcm_load` и добавьте опцию в `gr`:

```lua
{
    id  = "auto_close",
    type = "check",
    val = 1,
    def = true,
},
```

**Важно:** НЕ добавляйте `hint = "..."` — MCM сам добавит префикс `ui_mcm_` к значению hint и получится двойной префикс. Вместо этого пусть MCM автогенерирует `seqload_auto_close` и ищет `ui_mcm_seqload_auto_close`.

### Шаг 2. Объявить переменную модуля и загрузить значение

В том же файле вверху:

```lua
AUTO_CLOSE = true
```

В функции `load_settings()`:

```lua
AUTO_CLOSE = get_mcm_value("seqload/auto_close", true)
```

### Шаг 3. Добавить строки локализации

В `gamedata/configs/text/eng/st_sequential_load.xml`:

```xml
<string id="ui_mcm_seqload_auto_close">
    <text>Auto-close menu after saving preset</text>
</string>

<string id="ui_mcm_seqload_auto_close_desc">
    <text>When enabled, the preset configuration window closes automatically after you click Save.</text>
</string>
```

В `gamedata/configs/text/rus/st_sequential_load.xml`:

```xml
<string id="ui_mcm_seqload_auto_close">
    <text>Авто-закрытие меню после сохранения</text>
</string>

<string id="ui_mcm_seqload_auto_close_desc">
    <text>Если включено, окно настройки пресета закрывается автоматически после нажатия кнопки Сохранить.</text>
</string>
```

### Шаг 4. Перекодировать русский XML в CP1251

**ОБЯЗАТЕЛЬНО** после любой правки русского файла выполнить в PowerShell из корня мода:

```powershell
$p = 'gamedata\configs\text\rus\st_sequential_load.xml'
$t = [System.IO.File]::ReadAllText($p, [System.Text.Encoding]::UTF8)
[System.IO.File]::WriteAllText($p, $t, [System.Text.Encoding]::GetEncoding(1251))
```

### Шаг 5. Использовать значение в коде

```lua
if sequential_load_magazine_mcm.AUTO_CLOSE then
    sequential_load_magazine_ui.hide()
end
```

## 6. Как добавить кастомный дефолтный пресет для конкретного калибра

По умолчанию, если пресет не настроен, мод берёт **все** совместимые патроны в алфавитном порядке по 1 штуке. Но можно задать «встроенный по умолчанию» для конкретного калибра — например `{fmj x3, ap x1}` для 5.45x39.

### Вариант А (быстрый, через код)

В `sequential_load_magazine.script`, найдите функцию `build_default_preset(mag)` и перед алфавитной логикой добавьте:

```lua
-- Hardcoded per-caliber defaults (overrides the alphabetical fallback).
local HARDCODED_DEFAULTS = {
    ["5.45x39"] = {
        {ammo = "ammo_5.45x39_fmj", count = 3},
        {ammo = "ammo_5.45x39_ap",  count = 1},
    },
}

local function build_default_preset(mag)
    local canonical = get_mag_caliber(mag)
    if canonical and HARDCODED_DEFAULTS[canonical] then
        -- возвращаем копию, отфильтрованную по совместимости с конкретным магазином
        return filter_preset_for_mag(HARDCODED_DEFAULTS[canonical], mag)
    end
    -- ... existing alphabetical code ...
end
```

### Вариант Б (через сохранённый пресет)

Просто зайти в игру, открыть меню `U`, настроить пресет для нужного калибра, нажать Сохранить. Будет сохранён в текущий сейв.

**Минус:** надо делать в каждом новом сейве. Вариант А — глобально во все сейвы.

## 7. Как добавить новый контекстно-меню пункт на магазины

Сейчас есть один пункт — «зарядить поочерёдно». Если нужно добавить, например, «разрядить магазин в случайный патрон из инвентаря»:

### Шаг 1. Добавить строку перевода

В обоих XML:

```xml
<string id="st_seqload_random_unload">
    <text>разрядить случайно</text>
</string>
```
(не забыть перекодировать русский в CP1251 — см. шаг 4 выше)

### Шаг 2. Добавить функтор в конце `sequential_load_magazine.script`

```lua
local function random_unload_check(obj, bag, mode)
    -- ... проверки аналогично menu_check
end

local function random_unload_str(obj, bag, mode)
    return "st_seqload_random_unload"
end

local function random_unload_action(obj, bag, mode)
    -- логика разрядки
end

-- в on_game_start(), после существующей регистрации:
custom_functor_autoinject.add_functor("seqload_random_unload",
    random_unload_check, random_unload_str, nil, random_unload_action, true)
```

## 8. Как редактировать разметку UI-оверлея

Файл `gamedata/configs/ui/ui_sequential_load.xml`. Это обычный XML с размерами и координатами виджетов. В скрипте `sequential_load_magazine_ui.script` есть константы:

```lua
local MAX_ROWS   = 6
local ROW_HEIGHT = 44
```

Если нужно изменить количество строк в пресете или их высоту — правятся ОБА: `MAX_ROWS`/`ROW_HEIGHT` в Lua и тег `<row height="44" ... />` в XML синхронно. Иначе строки будут налезать друг на друга или будут пустоты.

## 9. Чек-лист перед коммитом правок

- [ ] Русский XML пересохранён в CP1251? (проверить через `[System.IO.File]::ReadAllBytes($p)[0..30]` — байты кириллицы должны быть в диапазоне 0xC0–0xFF, а не два байта 0xD0/0xD1 UTF-8 формата).
- [ ] Нет em-dash `—`, кавычек `« »`, длинного дефиса `–` в русском тексте (X-Ray может не рендерить — использовать ASCII `-` и `"`).
- [ ] В MCM-опциях нет явного `hint = "..."`.
- [ ] Для новых опций добавлены ОБЕ строки локализации: `ui_mcm_seqload_<id>` и `ui_mcm_seqload_<id>_desc`.
- [ ] Скрипты линтятся без ошибок (`ReadLints`).
- [ ] Проверено в игре: зашёл в MCM, вижу мод «Ammo Loader», опции с подписями на нужном языке, не raw-ключи.

## 10. Отладка: типичные проблемы

### «ui_mcm_menu_seqload» отображается как raw-ключ в списке MCM

→ Нет строки `ui_mcm_menu_seqload` в XML. Это ключ для названия в левом списке MCM. Добавить.

### «ui_mcm_ui_mcm_seqload_...» (двойной префикс)

→ В MCM-скрипте есть `hint = "ui_mcm_..."` у опции. Убрать весь параметр `hint`.

### Русские тексты отображаются как `????`

→ Файл сохранён не в CP1251. Выполнить команду перекодировки (см. шаг 4).

### Русские тексты отображаются как кракозябры `Р±СЂРёРІРµС‚`

→ Заголовок XML говорит `encoding="windows-1251"`, но байты в файле — UTF-8. То же лечится командой перекодировки.

### Отдельные русские символы отображаются как `?`

→ В тексте есть Unicode-символы, которых нет в CP1251 (em-dash `—` = U+2014, типографские кавычки `«»` = U+00AB/U+00BB в CP1251 ок, но шрифт может не поддерживать). Заменить на ASCII-эквиваленты (`-`, `"`).

### Патрон не появляется в выпадашке «Добавить тип патрона»

→ Скорее всего у этого патрона нет совместимых магазинов в Magazines Redux. Проверить через `magazine_binder.get_basetypes_by_ammo_type("ammo_xxx")` в консоли. Если возвращает `nil` — патрон игнорируется.

### Поочерёдная зарядка сразу прерывается

→ Обычно означает что кто-то другой держит action-slot Magazines Redux. Проверить: не открыт ли квик-реюз, не идёт ли reload оружия, не кликнут ли другой пункт в инвентаре сразу после.

### Счётчик магазина обновляется только в конце

→ Должен быть решён в коде. Если регресс — проверить что в `gradual_tick` вызывается `magazines.inventory_refresh(2)`, а НЕ `magazines.timed_refresh()`. Timed_refresh дебаунсит вызовы на 0.5s и при тиках каждые 0.4s никогда не срабатывает до конца.

---

## 11. Логирование

Для отладки своих правок добавляйте в скрипт:

```lua
printf("[seqload] some message %s", tostring(value))
```

Вывод увидите в `Anomaly\appdata\logs\xray_<user>.log`. Префикс `[seqload]` позволит быстро отфильтровать.
